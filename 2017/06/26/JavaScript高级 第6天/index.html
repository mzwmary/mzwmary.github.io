##2. 沙箱模式
一个隔离的环境

传参的目的

  1. 为了实现逻辑上的隔离，沙箱内部不使用外部的内容，外部也不使用沙箱内部的内容
  2. 便于代码压缩，因为内置对象无法被压缩，如果使用传参的方式，便可以压缩掉自己定义的这个形参

沙箱模式的用途

数组的方法

###1. forEach

  • 作用：遍历数组
  • 语法：arr.forEach(function(value,index,arr){ });
  • 参数：
  • value： 当前正在遍历的元素。
  • index： 当前正在遍历的元素的索引。

  • arr： 当前正在遍历的数组对象。

  • 返回值：undefined

forEach实现原理

###2. map

  • 作用：遍历数组
  • 语法：arr.map(function(value,index,arr){ });
  • 参数：
  • value： 当前正在遍历的元素。
  • index： 当前正在遍历的元素的索引。

  • arr： 当前正在遍历的数组对象。

  • 返回值：遍历数组时，循环调用回调函数，并每次将回调函数的返回值添加到数组中，最后返回数组。

map实现原理

###3. filter

  • 作用：遍历数组
  • 语法：arr.filter(function(value,index,arr){ });
  • 参数：
  • value： 当前正在遍历的元素。
  • index： 当前正在遍历的元素的索引。
  • arr： 当前正在遍历的数组对象。
  • 返回值：遍历数组时，每次调用回调函数，若回调函数的返回值是true,就将当前元素加入到返回值数组中。

filter实现原理

上下文调用模式

上下文调用模式中可以改变函数中this的指向！

  • lvalue: 左值，能够在等号左边接受别人赋值的值就可以称为左值。
  • rvalue: 右值，能够在等号右边给别人赋值的值就可以称为右值。
  • this是一个纯右值，只能用来给别人赋值，或者获取内容使用，不能被赋值！

    注意事项：

  1. 当给call和apply方法的第一个参数传递的是一个值类型的数据的时候，系统会自动将其抓换成对应的引用类型的数据，然后将this指向这个引用类型的数据。
  2. 如果给call和apply方法传递的第一个参数是null或者undefined，或者，一个参数都不传，函数中this会指向window对象！
